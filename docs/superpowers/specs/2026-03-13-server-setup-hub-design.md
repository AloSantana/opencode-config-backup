# Server Setup Hub - Design Specification

**Date:** 2026-03-13  
**Status:** Draft  
**Author:** Sisyphus (OpenCode AI)

---

## Executive Summary

Transform the `opencode-config-backup` repository into a comprehensive **Server Setup Hub** — a single source of truth for installing OpenCode, AI agents, modern development tools, and remote access capabilities. The system must be:

1. **LLM-installable**: An AI agent can read the repo and execute complete setup from a single prompt
2. **Wizard-first**: Interactive setup handles complexity (API keys, choices) before installation
3. **Modular**: Separate scripts for each tool category, enabling selective installation
4. **Universal**: Works on Ubuntu/Debian/macOS, with cloud VM optimizations

---

## Problem Statement

### Current State
- Repository contains OpenCode configs and basic installation script
- Documentation exists but is fragmented (INSTALLATION_HUB.md, GOOGLE_CLOUD_VM_SETUP.md)
- No interactive setup for API keys and environment configuration
- Missing 2024-2026 tools (new AI frameworks, MCP servers, DevOps tools)
- No Android remote access documentation

### Desired State
- **Single entry point** for LLM agents: `./install-all.sh --auto`
- **Interactive wizard** for human users: `./setup-wizard.sh`
- **Comprehensive tool catalog**: All modern dev tools, AI frameworks, MCP servers
- **Android remote access**: Complete guide for controlling server from mobile
- **Modular architecture**: Install only what you need

## Component Design

### 1. Setup Wizard (`setup-wizard.sh`)

**Purpose:** Interactive configuration before installation

**Responsibilities:**
- Detect environment (cloud VM, local, hybrid)
- Collect user preferences (tool categories, features)
- Validate and collect API keys
- Generate configuration files
- Call `install-all.sh` automatically

**User Flow:**
```
Welcome to OpenCode Server Setup Hub!

1. Environment Detection
   ✓ Detected: Ubuntu 24.04 on Google Cloud VM
   
2. Tool Selection
   Which categories do you want to install?
   [x] Base System (required)
   [x] Modern CLI Tools (recommended)
   [ ] DevOps Tools (Docker, k3s, CI/CD)
   [x] AI Frameworks (LangChain, Dify, etc.)
   [x] OpenCode Ecosystem (required)
   [ ] Android Remote Access
   
3. API Keys Configuration
   Enter your API keys (or press Enter to skip):
   • GitHub Token: ghp_***
   • Tavily API Key: tvly_***
   • OpenRouter API Key: sk-or-***
   [Skip remaining keys]
   
4. Generating Configuration
   ✓ Created ~/.config/opencode/.env
   ✓ Created config/manifest.json
   ✓ Created config/opencode.json
   
5. Starting Installation
   Calling install-all.sh...
```

**Generated Files:**
- `~/.config/opencode/.env`: Environment variables with API keys
- `config/manifest.json`: Installation plan (which categories, which tools)
- `config/opencode.json`: OpenCode configuration with MCP servers

---

### 2. Master Orchestrator (`install-all.sh`)

**Purpose:** Coordinate installation of all selected components

**Responsibilities:**
- Read `config/manifest.json` (or use defaults in `--auto` mode)
- Validate system requirements
- Call category scripts in dependency order
- Handle errors and provide rollback
- Run post-install verification
- Generate installation summary

**Execution Modes:**

**Interactive Mode** (called by wizard):
```bash
./install-all.sh
# Reads config/manifest.json
# Shows progress with user-friendly output
```

**Auto Mode** (for LLM agents):
```bash
./install-all.sh --auto
# Uses default manifest or pre-configured manifest
# Non-interactive, suitable for automation
```

**Selective Mode** (manual override):
```bash
./install-all.sh --categories="base,cli,opencode"
# Install only specified categories
```

**Installation Order:**
1. `install-base.sh` (always first - dependencies)
2. `install-cli.sh` (if selected)
3. `install-devops.sh` (if selected)
4. `install-ai.sh` (if selected)
5. `install-opencode.sh` (always - core functionality)
6. `install-android.sh` (if selected)
7. `verify-install.sh` (always last)

### 3. Category Scripts

Each category script follows this pattern:

```bash
#!/bin/bash
set -e

# Logging
LOG_FILE="$HOME/.opencode-setup.log"
log() { echo "[$(date +'%Y-%m-%d %H:%M:%S')] $1" | tee -a "$LOG_FILE"; }

# OS Detection
detect_os() { ... }

# Tool Installation Functions
install_tool_name() {
    if command -v tool_name &> /dev/null; then
        log "✓ tool_name already installed"
        return 0
    fi
    
    log "Installing tool_name..."
    # Installation commands
    log "✓ tool_name installed"
}

# Main
main() {
    log "=== Starting Category Installation ==="
    detect_os
    install_tool_1
    install_tool_2
    # ...
    log "=== Category Installation Complete ==="
}

main "$@"
```

**Key Features:**
- **Idempotent**: Check if tool exists before installing
- **Logged**: All output to `~/.opencode-setup.log`
- **Error handling**: `set -e` stops on first error
- **OS-aware**: Adapts commands for Ubuntu/Debian/macOS

2. **User Experience**: Human users complete wizard in < 5 minutes, full installation in < 30 minutes
3. **Modularity**: Users can install only desired categories

## Risk Mitigation

### Risk 1: API Key Management
**Mitigation:** 
- Wizard validates keys before proceeding
- Keys stored in `~/.config/opencode/.env` (not in repo)
- Template file shows required format

### Risk 2: Installation Failures
**Mitigation:**
- Idempotent scripts (safe to re-run)
- Detailed logging to `~/.opencode-setup.log`
- Verification script catches issues early

### Risk 3: Platform Compatibility
**Mitigation:**
- OS detection in each script
- Fallback installation methods
- Clear error messages for unsupported platforms

### Risk 4: Tool Version Conflicts
**Mitigation:**
- Pin versions where critical
- Document known conflicts
- Provide update script for safe upgrades

    "base": true,

export GITHUB_TOKEN="ghp_your_token"
